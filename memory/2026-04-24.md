# Memory — April 24 2026

## The Office — Discord Activity Project

### Context
- **Project:** The Office — multiplayer virtual office inside Discord (inspired by Gather.town)
- **Team:** 10 people (PM, 2 Devs, 5 Tech Artists, 2 Artists)
- **Tech Stack:** Node.js + Express + WebSocket + Canvas 2D (pixel art)
- **Hosting:** Railway (WebSocket server) + GitHub Pages (static)

### Architecture

**Server (`server/index.js`)**
- Express + WebSocket (ws) server
- Rate limiter: 100 msgs/min per connection
- Session persistence: players can reconnect with sessionId
- Input sanitization: XSS protection on name/text
- Structured JSON logging with timestamp, level
- Graceful error handling (uncaught exceptions)
- Health/stats endpoints
- Proximity-based chat (3 tile radius)

**Client (`client/index.html`)**
- Canvas 2D rendering
- Pixel art sprites (procedurally drawn)
- 8 color palettes for players
- Walk animation (4-frame cycle)
- Status system (available/focus/meeting/break)
- Proximity indicators (rings)
- Auto-hide chat with transition
- Reconnect with exponential backoff

**Zones**
- Open Office (10x8) — work area
- Focus Room (4x4) — silence
- Cafeteria (6x8) — social
- Arcade (8x6) — minijuegos
- Estudio (6x6) — presentations

### Key Fixes Applied
1. Keys + chat input conflict — solved with focus check
2. Movement throttle — 55ms minimum between moves
3. Zone boundaries — clamp to world limits
4. Ghost players — heartbeat with 30s timeout
5. Session persistence — reconnects preserve identity
6. XSS protection — sanitized inputs
7. Rate limiting — 100 msgs/min cap

### Test Suite
- `tests/game.test.js` — 7 automated tests
- Covers: health, stats, WS connect, join, move, chat, status

### Lessons Learned
- Railway doesn't support persistent WebSockets → use WebSocketServer properly with heartbeat
- Canvas pixel art needs careful coordinate math
- Proximity chat needs audio level indicators
- Status needs frontend + backend sync

### Next Steps
- Sprites pixel art from Artists
- Music per zone (embedded audio)
- Minijuegos in Arcade
- Discord Activity API integration
- Mobile responsive (canvas scaling)
